Crash Course on Regression Analysis

Concept 1: Linear Regression Basics

Linear regression is a statistical algorithm we use to model the relationship between a dependent variable y and one or more independent variables x

Singular Linear Regression

For a single independent variable, the model is:

y=β0+β1x1+ϵy = β_0 + β_1x_1 + \epsilony=β0​+β1​x1​+ϵ

Where:

  • β0β_0β0​ = the intercept of the regression line.
  • β1β_1β1​ = the slope of the regression line.
  • ϵ\epsilonϵ = the error term capturing deviations from the model.

Multiple Linear Regression

For multiple independent variables, the model generalizes to:

y=β0+β1x1+β2x2+⋯+βnxn+ϵy = β_0 + β_1x_1 + β_2x_2 + \dots + β_nx_n + \epsilony=β0​+β1​x1​+β2​x2​+⋯+βn​xn​+ϵ

Here, x1,x2,…,xnx_1, x_2, \dots, x_nx1​,x2​,…,xn​ are the predictors, and β1,β2,…,βnβ_1, β_2, \dots, β_nβ1​,β2​,…,βn​ are their corresponding coefficients.

Concept 2: Estimating Coefficients in Linear Regression

Slope and Intercept

The Linear Regression Algorithm estimates coefficients using the least squares method, minimizing the sum of squared errors ( ϵ\epsilonϵ^2 + ϵ\epsilonϵ^2 + ϵ\epsilonϵ^2 ) between real & predicted values. Since we only have a data set & not a formula, we cannot get the real value of the slope β1β_1β1​ - but we can obtain the estimated value β^1\hatβ_1β^​1​ (pronounced beta hat 1)

Concept 3: Non Linear Functions

Linear regression works for linear relationships, but what if we find a non-linear pattern along our adventures? For these cases, we use polynomial regression - a generalized form of linear regression. This involves fitting a curve, such as a quadratic (degree 2) or cubic (degree 3) equation, to the data.

Quadratic Regression

Quadratic regression models data as:
y=β0+β1x+β2x2+ϵy = β_0 + β_1x + β_2x^2 + \epsilony=β0​+β1​x+β2​x2+ϵ
The process involves adding a new feature x2x^2x2 (the square of the independent variable) and performing linear regression on the extended set of features.

Code Walkthrough for Quadratic Regression: 

def quadratic_regression(x, y):
        # add constant (intercept) and quadratic term (x^2) to the feature set
    X = sm.add_constant(np.column_stack((x, np.power(x, 2))))
    
        # fit an ordinary least squares (OLS) (linear) model
    model = sm.OLS(y, X).fit()
    
        # extract coefficients for intercept, linear, and quadratic terms
    intercept, linear, quadratic = model.params
    
        # calculate mean absolute error between actual and predicted values
    error = np.mean(np.abs(y - model.predict(X)))
    
        # return the error n formula
    return error, intercept + linear * age + quadratic * age**2

Process:

  1. X parameter engineering: We extend xxx to include both the original values and their squares (x2x^2x2).
  2. Model Fitting: The sm.OLS function performs Ordinary Least Squares regression using xxx and x2x^2x2 as features.
  3. Prediction: Predictions are made using the equation:
    y^=β0+β1x+β2x2\hat{y} = β_0 + β_1x + β_2x^2y^​=β0​+β1​x+β2​x2

In other words: Map [y,x][y, x][y,x] onto [y,x1,x2][y, x_1, x_2][y,x1​,x2​] where x1=xx_1 = xx1​=x & x2=x12x_2 = x_1 ^ 2x2​=x12​

Cubic Regression

Cubic regression extends this approach to include a third cubic term (x3x^3x3):
y=β0+β1x+β2x2+β3x3+ϵy = β_0 + β_1x + β_2x^2 + β_3x^3 + \epsilony=β0​+β1​x+β2​x2+β3​x3+ϵ

Code Walkthrough for Cubic Regression: 

def cubic_regression(x, y):
        # add quadratic term (x^2), and cubic term (x^3) to the feature set
    X = sm.add_constant(np.column_stack((x, np.power(x, 2), np.power(x, 3))))
    
        # fit lin model
    model = sm.OLS(y, X).fit()
    
        # extract coefficients for intercept, linear, quadratic, **and cubic** terms
    intercept, linear, quadratic, cubic = model.params
    
        # calculate mean absolute error between actual and predicted values
    error = np.mean(np.abs(y - model.predict(X)))
    
        # return error and the cubic prediction formula
    return error, intercept + linear * age + quadratic * age**2 + cubic * age**3

Explanation: xxx is extended to include x2x^2x2 and x3x^3x3 as features -- then we regress on all the parameters

Key Takeaways:

  • Polynomial regression uses higher-degree terms (x2,x3,…x^2, x^3, \dotsx2,x3,…) to capture non-linear relationships.
  • Quadratic regression adds one new feature (x2x^2x2), while cubic regression adds two (x2x^2x2 and x3x^3x3).
  • Both methods rely on the same principles as linear regression, with the added complexity of managing more predictors.

This framework is flexible and can be extended to higher degrees, though increasing the polynomial degree too much risks over-fitting.

Concept 4: The Error Term (ϵ\epsilonϵ)

Error Expectations on Linear functions

In a typical linear regression on a linear function, the error term ϵ\epsilonϵ is assumed to satisfy 2 conditions:

  1. Expected Value: The expected value of ϵ\epsilonϵ is zero (E[ϵ]=0E[\epsilon] = 0E[ϵ]=0).
    -- This is because E[ϵ]=mean(ϵ)=0E[\epsilon] = mean(\epsilon) = 0E[ϵ]=mean(ϵ)=0
  2. Normality: The error term is normally distributied (ϵ∼N(0,σ2)\epsilon \sim N(0, \sigma^2)ϵ∼N(0,σ2)).

Error from Regression on Different Functions

For y=xy = xy=x:

  • The error ϵ\epsilonϵ has E[ϵ]=0E[\epsilon] = 0E[ϵ]=0 since the model perfectly fits y=xy = xy=x (thus the mean evaluates to 0)

However for y=x+sin⁡(x)y = x + \sin(x)y=x+sin(x):

  • The expected value of the error term ϵ\epsilonϵ is E[ϵ]=sin⁡(x)E[\epsilon] = \sin(x)E[ϵ]=sin(x), reflecting the sinusoidal component not captured by the linear model.
  • How do we reduce the error further? By applying a regression on the error term
